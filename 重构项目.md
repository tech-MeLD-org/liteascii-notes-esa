# LiteASCII 项目重构指南

> **模型**: Kimi 2.5  
> **执行模式**: Agent 集群协作  
> **目标**: 全面 Tailwind CSS 化 + 架构目录重构

---

## 🎯 重构目标

1. **全面 Tailwind CSS 化**: 移除所有内联 `<style>` 标签，迁移到 Tailwind 工具类
2. **架构目录规范化**: 建立清晰、可维护的目录结构
3. **组件标准化**: 统一组件命名、接口设计、样式规范
4. **类型安全**: 完善 TypeScript 类型定义

---

## 📁 当前目录结构问题分析

```
src/
├── components/           # ❌ 混乱：UI组件、图表组件、特效组件混在一起
│   ├── ascii/           # ❌ 子目录过深
│   ├── DirectoryGraph.svelte
│   ├── ObsidianGraph.svelte
│   ├── AsciiArt.svelte
│   ├── Navigation.svelte
│   └── ...
├── pages/               # ❌ 页面与布局混合
├── layouts/             # ❌ 只有一个 Layout.astro
├── styles/              # ❌ 只有 global.css
├── utils/               # ✅ 尚可
└── content/             # ✅ 内容目录，保持不动
```

**核心问题**:
- 组件职责不清
- 样式分散在各文件的 `<style>` 中
- 缺乏统一的设计系统
- 图表组件和 UI 组件混在一起

---

## 🏗️ 目标目录结构

```
src/
├── components/                   # UI 组件层
│   ├── ui/                      # 基础 UI 组件 (Button, Card, Badge...)
│   ├── layout/                  # 布局组件 (Header, Footer, Sidebar, Nav)
│   ├── features/                # 功能组件 (NoteCard, TagCloud, CategoryList)
│   └── graph/                   # 图谱可视化组件 (D3.js 相关)
│
├── lib/                         # 工具库
│   ├── utils/                   # 工具函数
│   ├── hooks/                   # Svelte 自定义 hooks
│   └── constants/               # 常量定义
│
├── styles/                      # 全局样式
│   ├── global.css               # Tailwind 导入 + 基础样式
│   ├── components.css           # 组件级自定义样式
│   └── theme.css                # CSS 变量主题
│
├── types/                       # TypeScript 类型定义
│   ├── note.ts                  # 笔记相关类型
│   ├── graph.ts                 # 图谱相关类型
│   └── index.ts                 # 类型导出
│
├── content/                     # ✅ 保持现有内容结构
├── layouts/                     # Astro 布局
├── pages/                       # Astro 页面
└── assets/                      # 静态资源
```


### Agent 2: 基础 UI 组件库
**职责**: 将现有样式封装为 Tailwind 组件

**需要创建的组件**:

| 组件 | 路径 | 说明 |
|------|------|------|
| `Button` | `components/ui/Button.svelte` | 按钮组件 |
| `Card` | `components/ui/Card.svelte` | 卡片容器 |
| `Badge` | `components/ui/Badge.svelte` | 标签徽章 |
| `Pill` | `components/ui/Pill.svelte` | 圆形标签(Category/Tag) |
| `Icon` | `components/ui/Icon.svelte` | 图标组件 |

**样式迁移示例** (原 thoughts.astro 中的标签):
```svelte
<!-- 重构前 -->
<span class="meta-tag">#tag</span>
<style>
  .meta-tag {
    font-size: 0.7rem;
    color: var(--text-3);
    background: var(--bg-2);
    padding: 0.15rem 0.5rem;
    border-radius: 4px;
  }
</style>

<!-- 重构后 -->
<Pill variant="tag" size="sm">#tag</Pill>
```

**验收标准**:
- 所有 UI 组件使用 Tailwind 类
- 支持 `variant`, `size` 等 props
- TypeScript 类型完整

## 📝 Tailwind 设计规范

### 颜色系统
```js
// tailwind.config.mjs
colors: {
  primary: {
    DEFAULT: '#e74c3c',
    light: '#ff6b5b',
    dark: '#c0392b',
  },
  background: {
    DEFAULT: '#161618',
    secondary: '#1e1e20',
    card: '#242426',
  },
  text: {
    DEFAULT: '#e8e8e6',
    secondary: '#a8a8a6',
    muted: '#565658',
  },
  border: {
    DEFAULT: '#303032',
    light: '#424244',
  }
}
```

### 字体规范
```js
fontFamily: {
  sans: ['ui-sans-serif', 'system-ui', '-apple-system', 'sans-serif'],
  mono: ['"Courier New"', 'Courier', 'monospace'],
}
```

### 间距规范
使用 Tailwind 默认间距，或自定义:
```js
spacing: {
  '18': '4.5rem',
  '88': '22rem',
}
```
