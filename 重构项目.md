
# 角色与目标

你是一个资深的前端架构师（精通 Astro, Svelte 5 Runes, Tailwind CSS）。我需要你帮我重构一个名为「LiteASCII」的个人知识库项目。

目前的痛点：

1. **目录与逻辑混乱**：存在功能重复的文件和组件，比如 `src/utils` 和 `src/lib/utils` 职责不清，组件库中存在多个样式相近但未统一的卡片/标签组件。
2. **样式割裂**：代码中混合了大量的原生 `<style>` CSS 和零散的 Tailwind 类名。
3. **缺少全局规范**：相似的交互动画（如卡片悬停、标签高亮）散落在各个组件内部，没有实现“一处修改，全站同步”。

# 重构核心原则与要求

## 1. 目录结构大重组（请严格按此规划）

请帮我梳理并整合冗余文件，目标目录结构应如下：

- `src/lib/core/` (统一接管所有逻辑)
  - 将原来的 `src/utils/note-logic.ts` 升级为核心，整合 `src/lib/utils/note.ts` 中的重复功能（如路径解析、slug 转换、前后端笔记数据过滤等）。删除冗余的旧 utils 文件夹。
- `src/components/` (按原子化/功能分类)
  - `/layout/`: `Header.svelte`, `Footer.svelte`, `Sidebar.svelte`, `Navigation.svelte`, `PageHeader.svelte`
  - `/ui/`: `Pill.svelte` (取代原来的 `Tag.svelte` 和各种硬编码标签), `Card.svelte`
  - `/features/`: `DirectoryGraph.svelte`, `ReferenceGraph.svelte`, `NoteCard.svelte`, `AsciiAnimation.svelte`
  - **要求**：坚决删除功能重叠的组件（如删除 `PostCard.astro`，全站统一使用 `NoteCard.svelte`；删除 `Tag.svelte`，统一使用 `Pill.svelte`）。

## 2. 全面 Tailwind 化与全局复合语义类

彻底废弃组件内部的 `<style>` 标签（除了一些无法用 Tailwind 实现的极特殊场景）。
**核心要求**：在 `src/styles/interactions.css` 中，利用 Tailwind 的 `@layer components` 定义复合语义类，实现非标准交互的全局统一。
例如：

```css
@layer components {
  /* 全局统一的标签样式与悬停高亮 */
  .ui-pill {
    @apply inline-flex items-center px-2 py-0.5 rounded text-xs border transition-all duration-200;
  }
  .ui-pill-tag {
    @apply bg-bg-secondary text-text-muted border-border hover:border-primary/50 hover:text-primary;
  }
  /* 卡片联动高亮（当鼠标悬停卡片时，内部标签联动变色） */
  .interactive-card:hover .ui-pill-tag {
    @apply bg-primary-soft text-primary border-primary/30;
  }
}
```


## 3. Svelte 5 最佳实践

保持使用 Svelte 5 的 `$props()`, `$state()`, `$derived()`, 和 `{#snippet}` 语法，避免回退到 Svelte 4。
